서보 제어 명령어
|  |  |  |  |  | 
| SV1:<angle> |  | angle | SV1:90 | OK SV1=90 SV2=... | 
| SV2:<angle> |  | angle | SV2:45 | OK SV1=... SV2=45 | 
| R |  | 'R' | R | L,R | 
| <char> | wsadx |  | w | 0xA1 | 
| SPD:<0-255> |  |  | SPD:150 | OK SPD=150 | 
|  | [0x10][servoId][angle] |  | 0x10 0x01 0x5A |  | 



사용 예시 UDP 텍스트
- 서보1 120도: UDP로 SV1:120 전송.
- 서보2 30도: UDP로 SV2:30 전송.
- 센서값 요청: UDP로 R 전송 → ESP32가 Mega에 R 전송, Mega가 2바이트(왼,오)로 응답 → ESP32가 UDP 송신자에게 전달.

바이너리 명령(짧고 빠름)
- 포맷 제안: [0x10][servoId][angle]
- 0x10 = 서보 명령 헤더
- servoId = 0x01 또는 0x02
- angle = 0–180 (1바이트)
- 장점: 패킷이 작아 지연/오버헤드 감소.
- 주의: 송수신 쪽 모두 바이너리 규격을 동일하게 구현해야 함.

응답 및 신뢰성
- 텍스트 명령: ESP32는 처리 후 OK ... 형태로 응답을 보낼 수 있음.
- 바이너리 명령: 1바이트 ACK(예: 0xA0 성공, 0xFF 실패) 권장.
- UDP 특성: UDP는 비신뢰성 프로토콜이므로 중요한 명령은 ACK/재전송 로직을 추가하세요.

주의사항
- 전원: 서보는 별도 5V 전원 권장. ESP32 3.3V로 여러 서보 구동 금지. 공통 GND 필수.
- 각도 제한: 서보에 따라 권장 각도 범위가 다릅니다. 안전을 위해 0–180 범위를 좁혀 사용하세요.
- 명령 충돌: 모터 명령과 서보 명령이 동시에 들어올 경우 우선순위를 정하거나 큐로 처리하세요.
- 테스트: 먼저 작은 각도 변화(예: ±10°)로 테스트한 뒤 전체 범위를 확인하세요.
