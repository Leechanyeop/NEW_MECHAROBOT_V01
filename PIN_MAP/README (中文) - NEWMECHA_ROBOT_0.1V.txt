README (中文) - NEWMECHA_ROBOT_0.1V
系统模块
- MOTOR DRIVE – 机器人驱动与运动
- SENSOR_recog – 传感器识别
- motion_track – 障碍物检测时的运动控制
- CRC_tel – 服务器与机器人之间的通信节点
- vision_CAM – 视觉摄像头进行物体识别
- QR_visible – QR码识别与坐标计算

通信流程
服务器 → ESP32
- 目的路径设置
示例：
- 从 A 到 B
- 从 A 到 B 并在途中经过 C
- 到达 B 后经过 D，再返回 A
数字地图示例:

    _______________ B
A --------|---------------- C
    |_______________ D
- ESP32 读取服务器发送的 QR 值，并将其转换为实时导航路径。
- 服务器接收来自机器人的共享数据，包括传感器读数和障碍物检测视频。

ESP32 → 服务器
- 在每个路径分支处，ESP32 识别 QR 码并将其值发送到服务器。
- 当检测到障碍物时，ESP32 使用视觉和超声波传感器进行检查，并传输实时视频。
- ESP32 将机器人的各种传感器数据发送回服务器。

本文件清晰地概述了 NEWMECHA_ROBOT_0.1V 系统、其模块以及服务器与 ESP32 之间的通信工作流程。
