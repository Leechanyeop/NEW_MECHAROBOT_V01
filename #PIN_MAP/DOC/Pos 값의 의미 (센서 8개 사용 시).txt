Pos 값의 의미 (센서 8개 사용 시)
0: 라인이 가장 왼쪽 (첫 번째 센서) 아래에 있음
3500: 라인이 정확히 중앙에 있음
7000: 라인이 가장 오른쪽 (여덟 번째 센서) 아래에 있음
자세한 구간별 값
0 ~ 1000: 1번 센서와 2번 센서 사이
1000 ~ 2000: 2번 센서와 3번 센서 사이
...
6000 ~ 7000: 7번 센서와 8번 센서 사이
이 값을 PID 제어의 입력으로 사용하여, 3500이 되도록(중앙으로 오도록) 모터를 조절하게 됩니다.

최소값: 0 (라인이 가장 왼쪽 센서 아래에 있을 때)
최대값: 4000 (라인이 가장 오른쪽 센서 아래에 있을 때)
중앙값: 2000 (라인이 정확히 가운데 있을 때)
참고:

만약 센서가 라인을 완전히 벗어나면, 마지막으로 감지
했던 방향의 끝값(0 또는 4000)을 유지합니다

기준값 (목표): 2000 (라인이 정확히 중앙에 있을 때의 값)
현재값: 센서에서 읽어온 position (0 ~ 4000)
작동 원리:
linetrace.ino
 코드 112번 줄: int16_t error = 2000 - position;
2000에서 현재 위치를 뺀 값(error)을 0으로 만드는 것이 목표입니다.
만약 position이 1000이면, error는 +1000이 되고, 이만큼 왼쪽 모터를 줄이고 오른쪽 모터를 키워서 다시 2000(중앙) 쪽으로 회전하게 됩니다.
결국 **"2000(중앙)을 기준으로, 벗어난 만큼 되돌아오게 작동한다"**고 보시면 됩니다.
