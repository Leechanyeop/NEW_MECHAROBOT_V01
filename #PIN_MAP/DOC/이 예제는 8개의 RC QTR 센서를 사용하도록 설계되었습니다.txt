// 이 예제는 8개의 RC QTR 센서를 사용하도록 설계되었습니다.
// 이 반사 센서들은 디지털 핀 3번부터 10번까지 연결해야 합니다.
// 센서의 발광 제어 핀(CTRL 또는 LEDON)은 선택적으로 디지털 핀 2번에 연결할 수 있으며,
// 연결하지 않을 경우 setEmitterPin() 호출을 제거하면 됩니다.
//
// 예제의 setup 단계에서는 센서를 10초 동안 보정(calibration)하며,
// 이 동안 아두이노의 LED(보통 핀 13번)가 켜집니다.
// 보정 단계에서는 각 반사 센서가 가장 밝은 값과 가장 어두운 값을 측정할 수 있도록 해야 합니다.
// 예를 들어 라인 트레이서를 만들 경우, 보정 단계에서 센서를 라인 위로 움직여서
// 센서가 라인의 어두운 부분과 바닥의 밝은 부분을 모두 읽을 수 있도록 해야 합니다.
// 올바르지 않은 보정은 잘못된 측정 결과를 초래합니다.
//
// 메인 루프에서는 보정된 센서 값을 읽고 이를 이용해 라인의 위치를 추정합니다.
// 이를 테스트하려면 흰 종이에 3/4인치 검은 절연 테이프를 붙이고
// 센서를 그 위로 움직여 보시면 됩니다.
// 시리얼 모니터에는 센서 값이 출력되며,
// 값은 0(최대 반사율)에서 1000(최소 반사율)까지의 숫자로 표시됩니다.
// 이어서 라인의 추정 위치가 0에서 5000까지의 숫자로 출력됩니다.
// 예를 들어 1000은 라인이 센서 1 바로 아래에 있다는 의미이고,
// 2000은 센서 2 바로 아래에 있다는 의미입니다.
// 0은 라인이 센서 0 바로 아래에 있거나, 센서 0에서 마지막으로 감지된 후 사라졌음을 의미합니다.
// 5000은 라인이 센서 5 바로 아래에 있거나, 센서 5에서 마지막으로 감지된 후 사라졌음을 의미합니다.
