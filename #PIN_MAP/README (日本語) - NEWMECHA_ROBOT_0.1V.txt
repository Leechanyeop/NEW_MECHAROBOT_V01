README (日本語) - NEWMECHA_ROBOT_0.1V
システムモジュール
- MOTOR DRIVE – ロボットの走行と移動
- SENSOR_recog – センサー認識
- motion_track – 障害物検知時の動作制御
- CRC_tel – サーバーとロボット間の通信ノード
- vision_CAM – ビジョンカメラによる物体認識
- QR_visible – QRコード認識と座標計算

通信フロー
サーバー → ESP32
- 目的地経路設定
例：
- A から B へ移動
- A から B へ移動し、途中で C に立ち寄る
- B に到着後、D を経由して A に戻る
デジタルマップ例:

    _______________ B
A --------|---------------- C
    |_______________ D
- ESP32 はサーバーから送られた QR 値を読み取り、リアルタイムのナビゲーション経路に変換する。
- サーバーはロボットから共有されるセンサー情報や障害物検知映像を受信する。

ESP32 → サーバー
- 各経路分岐で、ESP32 は QR コードを認識し、その値をサーバーへ送信する。
- 障害物を検知した場合、ESP32 はビジョンセンサーと超音波センサーで確認し、リアルタイム映像を送信する。
- ESP32 はロボットの各種センサー値をサーバーへ送信する。

この文書は NEWMECHA_ROBOT_0.1V システム、そのモジュール、そしてサーバーと ESP32 間の通信ワークフローを明確に概説しています。
